# Archive

This folder keeps documentation that remains useful for history or traceability but should not be treated as active implementation guidance.

- `legacy/` contains superseded plans, migration notes, and architecture writeups.
- `legacy/PRD.md` contains the original monolithic repository PRD retained only as historical baseline context.
- `handoffs/` contains historical handoff notes kept only for traceability.
- `reports/` contains generated audits, refactor reports, and analysis output.

If a document starts driving current implementation again, move it back into an active docs area and update `docs/README.md`.
