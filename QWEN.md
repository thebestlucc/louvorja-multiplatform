# LouvorJA Multiplatform — QWEN Context

## Project Overview

**LouvorJA Multiplatform** is a cross-platform church worship management desktop application built with **Tauri 2**, **React 19**, and **Rust**. It is a migration from a legacy Delphi version to run on Windows, macOS, and Linux.

### Core Features
- **Hymnal** — Browse, search (FTS5), and project hymn lyrics with multi-slide display
- **Presentation Editor** — Create/edit slide decks with drag-and-drop, import `.pptx` files
- **Audio Playback** — Synchronized audio via rodio engine
- **Bible** — Full-text search across Bible versions with verse projection
- **Worship Services** — Service scheduling with drag-and-drop timeline
- **Multi-Monitor** — Projector, return monitor, per-monitor assignment
- **Live Streaming** — HTTP server with SSE for remote viewers
- **Archive Format** — Custom `.slja` ZIP-based archive for presentations
- **Auto-Updates** — Service-aware update guard
- **Internationalization** — Portuguese, English, Spanish (i18next)
- **Themes** — 5 color themes via CSS custom properties

## Key Commands

```bash
# Dependencies
pnpm install              # Install all deps

# Development
pnpm dev                  # Vite dev server only (no Rust)
pnpm dev:main             # Full dev mode (tauri dev = frontend + Rust hot-reload)
pnpm tauri dev            # Same as dev:main

# Build
pnpm build                # TypeScript check + Vite build
pnpm tauri build          # Production build (all platforms)

# Checks
npx tsc --noEmit          # TypeScript type check only
cargo check --manifest-path src-tauri/Cargo.toml  # Rust check only
pnpm test:unit            # Run unit tests (Node native test runner)

# IMPORTANT: After adding routes, run `pnpm vite build` BEFORE `npx tsc --noEmit`
# The Vite plugin generates routeTree.gen.ts — stale types cause TS errors.
```

### Package Manager
- **Always use `pnpm`** for frontend operations. Never use `npm` or `deno`.
- Use `pnpm tauri add <plugin>` for Tauri plugins (auto-registers in Cargo + permissions).

## Architecture Patterns

### Rust Backend
- **Error handling**: All functions return `Result<T, AppError>`. Use `Err(AppError::Internal("msg"))` for stubs — never `todo!()` (panics crash the app).
- **Commands**: Each `#[tauri::command]` takes `state: tauri::State<'_, AppState>`, gets a connection from the `r2d2` pool, delegates to `db::queries::*`.
- **Registration**: Commands registered via `tauri_specta::collect_commands!` in `lib.rs` (NOT `tauri::generate_handler!`).
- **Database**: SQLite via rusqlite with r2d2 pool. WAL mode + foreign keys enabled.
- **Threading**: All long-running operations must use `std::thread::spawn` — blocking ops in commands hang IPC (especially on Windows).
- **Serde**: Use `#[serde(rename_all = "camelCase")]` on structs returned to frontend.

### Frontend
- **Bindings**: Import types from `@/lib/bindings` (auto-generated) — never define manual interfaces for backend data.
- **Error handling**: Use `catcher.ts` — `const [data, err] = await catcher(promise, { notify: true })`. Do NOT use manual try-catch.
- **Components**: Use CVA pattern with exported `*Variants` functions for style variants.
- **State**: Zustand stores for client-only UI state. Use `Store.getState()` inside async callbacks to avoid stale closures.
- **Navigation**: Always use TanStack Router `<Link to="/path">` — never `<a href>` or `window.location`.
- **i18n**: Always add keys to ALL THREE locale files (en, pt, es).

### New Tauri Commands Checklist
1. Add query in `db/queries/*.rs`
2. Add command in `commands/*.rs`
3. Register in `lib.rs` via `tauri_specta::collect_commands!`
4. **Add command name to `src-tauri/permissions/commands.toml`**
5. Add wrapper in `lib/tauri.ts` if needed
6. Add hook in `lib/queries.ts`

## Common Pitfalls

### Rust
- **quick-xml temporary lifetime**: Bind `e.name()` to a variable before calling `.as_ref()`
- **rodio OutputStream**: Wrap with `unsafe impl Send/Sync` + Mutex
- **Windows IPC**: Long-running ops must spawn a thread — blocking commands hang the entire IPC bridge
- **Windows path separators**: Always `.replace('\\', "/")` on paths sent to frontend
- **Audio init**: Initialize audio in a background thread with timeout — `OutputStream::try_default()` can block forever on Windows

### TypeScript/React
- **React 19 useRef**: Requires explicit initial value: `useRef<T>(undefined)`
- **Stale closures**: Use `Store.getState()` inside async callbacks
- **Dead key/IME composition**: Use local state + dirtyRef for input bound to server data
- **Conditional hooks**: Never call hooks after early return — use `{ enabled: false }` pattern

### General
- **Don't delete scaffolded code**: Check `docs/plans/` for planned features
- **i18n**: Keys missing in any locale render as raw key strings
- **Package manager**: Always `pnpm`, never `npm` or `deno`
